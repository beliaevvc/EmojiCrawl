# Game Domain Layer

Это ядро игровой логики.
Здесь живут правила, модели и чистые функции для управления состоянием игры.

## Правила архитектуры:
1. **Никакого React**: здесь не должно быть хуков, компонентов или JSX.
2. **Никакой Инфраструктуры**: прямые вызовы API, Supabase или LocalStorage запрещены.
3. **Чистые функции**: логика должна быть тестируемой и предсказуемой.
4. **Контент через интерфейсы**: в идеале, данные (карты/монстры) приходят извне, но на этапе Блока 1 допускаются импорты из `src/data`.

## Структура:
- `model/`: Типы данных (GameState, Card, Player).
- `reducer/`: Чистая функция редьюсера, обрабатывающая действия.
- `deck/`: Логика создания и перемешивания колоды.
