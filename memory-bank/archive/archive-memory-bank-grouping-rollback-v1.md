## IMPLEMENTATION
- Отказались от структурных перемещений документов.
- Зафиксировали вывод: порядок лучше достигать через:
  - индекс в `memory-bank/README.md`,
  - единый формат документов,
  - стабильные пути и перекрёстные ссылки.

## TESTING / VERIFICATION
- Ручная проверка навигации и ссылок после отката: пути снова соответствуют ожиданиям.

## LESSONS LEARNED
- Для `memory-bank/` важнее **стабильность путей** и **простота доступа**, чем иерархия.
- Улучшать UX документации стоит “сверху вниз”: индекс → соглашения → только потом (если нужно) физическая структура.
